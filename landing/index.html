✦ Features

Why teams use Brela

Whether you're a solo developer or a large team, Brela gives you complete visibility into your AI-assisted workflow.

👁️‍🗨️

Silent tracking

Zero config, zero friction. Brela works in the background through VS Code hooks, shell wrappers, and git integration.

🔌

Multi-tool support

Detects code from Copilot, Claude Code, Cursor, Windsurf, Cline, Aider, Continue, and ChatGPT paste — all at once.

🔒

Privacy-first

All attribution data stays on your machine. Nothing leaves your filesystem. The .brela/ folder is automatically gitignored.

📊

Instant reports

Run brela report for a full breakdown by tool, file, author, or date range. Drill into any file with brela explain.

🛡️

100% local. Zero telemetry.

Brela never phones home. All session data lives in .brela/ on your disk. We don't collect, store, or transmit any of your code or usage data.

⚡ Simple setup

How it works

One command to set up. Then it runs silently in the background.

1

Install & init

Run brela init in your project root. It sets up hooks, shell wrappers, and the VS Code extension.

2

Code normally

Use any AI tool — Copilot, Claude, Cursor—however you like. Brela detects and records every AI contribution silently.

3

Get reports

Run brela report for a full breakdown. See which AI wrote what, per file, per day, per author.

4

Deep dive

Use brela explain <file> to see the full attribution history for any specific file in your project.
